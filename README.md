## About

I am a backend engineer and systems programmer with a side obsession for speech intelligence. My days are spent designing APIs, refactoring legacy systems, and making services talk to each other. My nights are spent making computers understand human speech — without sending a single byte to the cloud.

These two worlds are not separate. Both require the same discipline: respect the user's resources, minimize latency, and never trust a network request you don't have to make.

I believe software should work where the user is, not where the vendor wants it to run. That principle applies whether I'm building a microservice architecture or a local STT engine.
